# Planning with Files Skill：AI Agent 持久化計劃系統

## 概述

Planning with Files 是一個持久化文件計劃系統，為 Claude Code 和其他 AI Agent 提供穩定、可靠的任務協調能力。通過將計劃保存到 Markdown 文件，即使在上下文丟失或會話中斷情況下，AI Agent 也能恢復進度並繼續執行任務。

**關鍵詞**：計劃管理、持久化計劃、Agent 協調、崩潰恢復、多 Agent 共享狀態

## 何時使用此技能

- 📋 需要長期運行的 Agent 任務
- 💾 防止上下文丟失導致進度丟失
- 🔄 多個 Agent 需要共享和協調計劃
- 🏗️ 複雜的多步驟項目管理
- 🛡️ 需要任務崩潰恢復能力
- 📊 追蹤和驗證 Agent 執行進度

## 核心功能

### 1. 持久化計劃存儲
- Markdown 格式的計劃文件
- 自動計劃更新和版本控制
- 上下文丟失後自動恢復

### 2. 確定性完成檢測
- 明確的完成條件定義
- 避免無限循環
- 智能退出判斷

### 3. 多 Agent 協作
- 磁盤上的共享狀態存儲
- Agent 間的計劃同步
- 並行執行支持

### 4. 任務進度追蹤
- 實時進度更新
- 詳細的執行日誌
- 完成度統計

## 計劃結構

### 標準計劃格式
```markdown
# 項目名稱

## 目標
明確定義最終目標

## 計劃步驟
- [ ] 步驟 1 - 具體描述
- [ ] 步驟 2 - 具體描述
- [ ] 步驟 3 - 具體描述

## 完成條件
描述完成項目的確切條件

## 進度追蹤
- 當前步驟: 步驟 2
- 完成度: 33%
- 最後更新: 時間戳
```

## 實施步驟

### 1. 初始化計劃
```bash
# 創建計劃文件
npx skills add othmanadi/planning-with-files
```

### 2. 定義計劃結構
```markdown
# 軟件開發項目計劃

## 目標
完成具有前端、後端、數據庫的完整應用

## 計劃步驟
- [ ] 設計系統架構
- [ ] 實現後端 API
- [ ] 構建前端界面
- [ ] 集成數據庫
- [ ] 編寫單元測試
- [ ] 部署到生產環境

## 完成條件
所有功能實現、所有測試通過、應用成功上線
```

### 3. 執行和追蹤
```python
# Agent 根據計劃執行
# 完成每個步驟後更新計劃
# - [ ] → - [x]

# 系統自動檢查完成條件
# 當所有步驟完成 → 項目結束
```

### 4. 恢復和續行
```bash
# 即使會話中斷，也能恢復
# 加載計劃文件，查看已完成的步驟
# 從當前進度繼續執行
```

## 最佳實踐

### ✅ 推薦做法
- 使用清晰、可驗證的步驟描述
- 定期更新計劃文件中的進度
- 為每個步驟定義明確的完成條件
- 在計劃中記錄關鍵決策和理由
- 使用 Markdown 清單 (- [ ]) 跟蹤進度
- 定期審查和調整計劃

### ⚠️ 注意事項
- 避免計劃過於詳細或過於簡略
- 確保完成條件可客觀驗證
- 不要頻繁修改已完成的步驟
- 保持計劃文件在項目根目錄易於訪問

## 常見問題

### Q: 會話中斷後如何恢復？
**A**: Planning with Files 將計劃保存到磁盤文件。重新開始時加載計劃文件，系統會自動顯示已完成的步驟和當前進度。

### Q: 能否用於真實的大型項目？
**A**: 是的，該系統被設計用於生產級別的複雜項目。許多開發團隊用它來協調多 Agent 的長期開發任務。

### Q: 如何避免 Agent 無限循環？
**A**: 通過定義明確的完成條件（deterministic completion gate）。Agent 必須滿足這些條件才能標記任務完成。

### Q: 多個 Agent 能否同時編輯同一計劃？
**A**: 支持，系統提供文件鎖定機制確保數據一致性。建議使用時間戳和 conflict resolution 策略。
