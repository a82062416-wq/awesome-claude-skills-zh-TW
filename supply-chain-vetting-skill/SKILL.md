# supply-chain-vetting

## 安裝前必查（逐項）
1. **來源**：作者是誰？star/使用數？最近更新？（棄坑專案風險高）
2. **權限**：要求哪些工具/網路存取？超出功能所需就可疑
3. **腳本審查**：hook 腳本、install script 逐行讀——有無外部網路請求、讀憑證、執行遠端代碼
4. **無法審查就不裝**：拿不到原始碼、只給編譯檔 → 拒絕

## 安全安裝法（本環境已驗證有效）
外部倉庫無法信任時，**讀規格 → 原生重寫**，而非直接 clone 執行。
（2026-07-07 Fable Harness 即用此法：WebFetch 審查後繁中重實作，供應鏈風險歸零）

## 紅旗
- 要求 curl | bash 一鍵安裝
- hook 腳本含 base64 解碼後執行
- 索取環境變數、SSH key、雲端憑證
