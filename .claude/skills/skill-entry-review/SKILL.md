# 技能條目審查

對指定的技能資料夾（或本次 diff 涉及的所有技能）執行以下檢查，輸出表格式審查報告。

## 檢查清單

1. **Frontmatter**：SKILL 檔案開頭必須有 `name` 與 `description`；name 與資料夾名一致
2. **README 索引**：`README.md` 與 `README.en.md` 都有這個技能的條目，且放在正確分類、描述一句話講清楚用途
3. **中英同步**：若有 `SKILL.zh-TW.md` 與 `SKILL.en.md`，兩份的章節結構與關鍵內容一致
4. **連結有效**：跑 `python3 scripts/check-links.py`，該資料夾相關連結零破損
5. **誠實原則**（最重要）：
   - 沒有虛構的效能數據（「10 倍加速」「99% 準確」需有出處，否則刪除）
   - 沒有不存在的 API、套件或 import 範例
   - 外部工具連結指向真實存在的專案
6. **風格**：無 AI 味套話（參考 stop-slop 技能）

## 輸出格式

| 檢查項 | 結果 | 問題 / 建議 |
|--------|------|------------|

結尾給一行結論：✅ 可合併 / ⚠️ 修正後合併 / ❌ 需重寫。
只報告，不自動修改；使用者同意後才動手修。
