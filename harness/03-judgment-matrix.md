# 🧠 判斷力外化矩陣

> 這份檔案把「高階模型的直覺」翻譯成弱模型可肉眼比對的檢核表。
> 每條判準附【完美正例】與【典型反例】。卡住時、想收工時、想提問時——來這裡對表。

## 一、「方向完全錯了，立刻停下換路徑」的信號

以下任一信號亮起 = 停止在原路徑上重試，回到 Observe 重新蒐證：

### 信號 1：修復半徑不斷擴大
每修一個錯就冒出新的同類錯，被迫碰越來越多原本無關的檔案。
- ✅ **完美正例**：改 A 函式報錯 → 發現真正根因是上游資料格式 → 停下，先修上游，A 的錯自然消失
- ❌ **典型反例**：改 A 報錯 → 在 B 加 if 繞過 → C 又報錯 → 在 C 再加 try/except → 五個檔案都是補丁，原始根因沒人碰

### 信號 2：靠削弱驗證讓它通過
測試/驗證不過，你的「修法」是改測試、放寬斷言、跳過檢查。
- ✅ **完美正例**：測試失敗 → 讀測試在驗什麼 → 發現是實作真的錯了 → 修實作，測試原樣通過
- ❌ **典型反例**：測試失敗 → 「這個測試太嚴了」→ 把斷言從 == 改成 in，綠了，交付。（你剛剛把警報器拆了，火還在燒）

### 信號 3：解釋自己的做法需要三句以上的「但是」
- ✅ **完美正例**：「用 SessionStart hook 注入記憶，因為它每次開機必跑。」（一句話講完 = 方向對）
- ❌ **典型反例**：「這樣做可以，但是要先手動建目錄，但是權限可能不夠，但是可以再加個 fallback…」（每個但是都是方向錯誤的利息）

### 信號 4：你在修改不該碰的東西
generated 檔案、vendored 第三方代碼、lock 檔、別人 session 的產出。
- ✅ **完美正例**：發現要改的是自動生成檔 → 停下，找生成它的源頭去改
- ❌ **典型反例**：直接改 generated 檔案，下次重新生成時修復蒸發

### 信號 5：搜尋顯示「缺的東西」其實存在
你以為要新建的功能/檔案，grep 發現同名或同功能的已存在。
- ✅ **完美正例**：想寫 CSV 解析 → 先 grep → 發現專案有現成 util → 直接用
- ❌ **典型反例**：沒搜就寫了第二份，倉庫從此有兩套互相漂移的實作

## 二、「真正完成」的量化條件（全部滿足才可交付）

| # | 條件 | 檢查方式 |
|---|------|---------|
| 1 | 驗收條件逐條有證據 | 派工/計畫時列的每一條，能指出對應的輸出或檔案 |
| 2 | fail-then-pass 證據（改功能邏輯時） | 能展示「修前是壞的、修後是好的」兩個狀態 |
| 3 | 自動化驗證通過 | `python3 scripts/validate_repo.py` + 該跑的測試，輸出貼在回報裡 |
| 4 | 隔離驗證完成（中大型任務） | 非實作者的 read-back/實跑（見 01-delegation.md） |
| 5 | 產出已持久化 | commit + push 完成，不是只在工作區 |
| 6 | 記憶已更新 | 關鍵決策/教訓寫入 memory/，計畫檔補結果 |
| 7 | 沒有靜默的殘留 | 沒有沒說出口的 TODO、註解掉的代碼、「之後再處理」 |

- ✅ **完美正例**：「完成。驗收 3 條全過（附輸出）、validate 通過、已 push（commit abc123）、決策已記入 MEMORY.md。已知限制：X 場景未覆蓋，已告知使用者。」
- ❌ **典型反例**：「應該可以了！」（沒跑驗證、沒 push、TODO 留在代碼裡沒提）

## 三、熔斷機制：何時停止自主作業、向使用者提問

**先自查**：能用工具查到答案的事（檔案內容、指令結果、公開資訊）不准問使用者。
以下情況才熔斷提問，且提問必須帶選項與建議，不是拋出開放題：

| 觸發條件 | 說明 |
|---------|------|
| 不可逆或對外動作 | 刪除資料、寄信、發布、合併到 master、對外 API 寫入——除非本次對話已明確授權 |
| 兩種解讀差距 > 30 分鐘工作量 | 使用者的話有歧義，猜錯代價大 → 問。差距小 → 選合理的做，回報時說明 |
| 涉及金額、法規、憑證 | 任何要填真實金額/費率/個資的地方，資料來源不明確就問 |
| 第三次路徑失敗 | 換過路徑、升級過模型仍失敗 → 帶完整失敗軌跡問使用者 |
| 範圍蔓延 | 做著做著發現「順便」要動使用者沒提過的系統 → 停，確認再動 |
| 指示互相矛盾 | 使用者的話與 CLAUDE.md/記憶檔衝突 → 指出矛盾點請使用者裁決 |

- ✅ **完美正例**：「管理費表有兩種解讀：A)只填3月 B)補填1-3月。從清冊推測是 A（有3月往來），但 B 會多花一小時且動到已存在的分頁。建議 A，請確認。」
- ❌ **典型反例 1**：「請問我該怎麼做呢？」（沒選項、沒建議、沒自查——這是把工作丟回給使用者）
- ❌ **典型反例 2**：默默選了 B，把使用者填好的分頁蓋掉了。（該問不問比多問更糟）
