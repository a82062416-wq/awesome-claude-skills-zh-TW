# 長期記憶系統（Git-based Memory）

## 為什麼用檔案 + Git 當記憶？

| 方案 | 跨 session | 跨裝置 | 費用 | 隱私 |
|------|-----------|--------|------|------|
| claude-mem 插件 | ✅ | ❌ 單機 | 免費 | ⚠️ 本機 API 無認證 |
| Mem0 / Supermemory | ✅ | ✅ | 雲端付費 | 資料在第三方 |
| **本系統（memory/ + git）** | ✅ | ✅ | 免費 | ✅ 只在自己的 repo |

雲端 Claude Code 的容器是暫時的，唯一可靠的持久層就是 git。
記憶檔 commit 後（本倉庫 hooks 會自動推送），任何裝置、任何 session 都讀得到。

## 記憶協定（Claude 必須遵守）

### 1. Session 開始 → 讀取記憶
```bash
cat memory/MEMORY.md
```
CLAUDE.md 已規定開場必讀。讀完就知道使用者是誰、偏好什麼、專案進行到哪。

### 2. 工作中 → 捕捉值得記的事
符合以下任一條件就該寫入記憶：
- 使用者說「記住」「以後都這樣」「我偏好…」
- 發現使用者的新事實（新專案、新物件名稱、新格式要求）
- 踩到坑學到教訓（某方法行不通、某設定才正確）
- 專案狀態變化（PR 開了/合了、技能新增了）

### 3. 寫入 → 更新對應區塊
- 👤 使用者輪廓：身分、工作領域相關的長期事實
- ⚙️ 使用偏好：格式、語言、流程偏好
- 📌 環境狀態：技能、連接器、設定的現況
- 📚 專案進行中：活躍任務與進度
- 💡 已學到的教訓：踩坑記錄

### 4. 維護原則
- **精簡**：只記下次 session 還需要的事，總量控制在 150 行內
- **汰舊**：過時的直接刪，不要累積（完成的專案移到一行總結）
- **立即 commit**：更新後馬上 `git add memory/ && git commit -m "🧠 更新記憶：<摘要>"`

### 5. 大事件 → 歸檔
重大專案結束時，把詳細過程寫到 `memory/archive/<專案名>.md`，
MEMORY.md 只留一行摘要 + 連結。

## 使用者常用指令

| 你說 | Claude 做 |
|------|----------|
| 「記住：…」 | 寫入 MEMORY.md 對應區塊並 commit |
| 「你記得什麼？」 | 朗讀 MEMORY.md 摘要 |
| 「更新記憶」 | 回顧本次對話，把值得記的寫入 |
| 「忘掉 X」 | 從 MEMORY.md 刪除該項並 commit |
