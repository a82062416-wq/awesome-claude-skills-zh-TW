# Stop Slop：去除 AI 生成文檔的"AI 味"

## 概述

Stop Slop 是一個內容優化技能，用於識別和清除 Claude 生成內容中常見的"AI 味"特徵。這些特徵包括：冗余的修飾詞、模板化表述、虛無的形容詞、過度正式的語氣，以及不必要的結構化標記。

**關鍵詞**：文檔優化、去除 AI 痕跡、語言簡化、專業文案、自然表達、內容潤色

## 何時使用此技能

- 📄 撰寫報告、郵件或文檔時，想讓文字更自然、更像人寫的
- ✏️ 需要潤色 Claude 生成的文本，移除明顯的 AI 特徵
- 📝 創建面向客戶或公開的內容，需要高度專業感
- 🎯 優化長篇文章、博客或技術文檔
- 💼 準備簡歷、求職信或商業提案
- 🔍 審查任何生成文本，確保質感和自然度

## "AI 味"的 10 大特徵與修復

### 1. ❌ 虛無修飾詞堆砌

**症狀**：過度使用空心詞彙
- "革命性的"、"創新的"、"領先的"、"獨特的"、"優質的"

**修復方案**：
```
❌ 我們提供革命性的解決方案
✅ 我們的方案將周期從 6 個月縮短至 2 週
```

**規則**：
- 移除所有無法用數據支持的修飾詞
- 用具體成果替代空心宣傳

### 2. ❌ 過度正式的開場白

**症狀**：
- "作為一個 AI 助手，我很高興..."
- "值得注意的是..."
- "在深入討論之前..."
- "首先，讓我們探討..."

**修復方案**：
```
❌ 作為一個 AI 助手，我很高興為您解釋...
✅ 以下是解釋...

❌ 值得注意的是...
✅ 要點是...（或直接陳述事實）
```

### 3. ❌ 模板化的結構標記

**症狀**：過度使用編號列表和符號
```
❌ 
## 概述
## 關鍵要點
## 實施步驟
1. 步驟 1
2. 步驟 2
...
## 結論
```

**修復方案**：
- 只在邏輯上需要時才使用列表
- 移除冗余的大標題（如果只有 1-2 項內容）
- 使用段落敘述來代替簡單列表

### 4. ❌ 過度句式轉移

**症狀**：連續出現過多相似句式結構
- "首先...其次...最後..."
- "一方面...另一方面..."
- "值得注意的是...同樣地..."

**修復方案**：
```
❌ 首先，設計很重要。其次，實現也很重要。最後，測試同樣關鍵。
✅ 好的設計需要經過實現和測試的驗證。
```

### 5. ❌ 不必要的認知過渡詞

**症狀**：過度使用轉移詞
- "值得注意的是"
- "總之"、"總的來說"
- "有趣的是"
- "需要強調的是"

**修復方案**：
- 移除 90% 的認知過渡詞
- 使用段落分割替代
- 讓想法本身連接，而不是靠詞語連接

### 6. ❌ 冗余的重複確認

**症狀**：相同想法重複表述
```
❌ 這很重要。值得強調的是，這確實很重要。簡而言之，重要性不可忽視。
✅ 這很重要。
```

**修復方案**：
- 一個想法只表述一次
- 移除所有"換句話說"式的重複

### 7. ❌ 虛無的問句開場

**症狀**：用修辭問句開場
```
❌ 有沒有想過...？
✅ ...（直接陳述）
```

**修復方案**：
- 移除所有修辭問句
- 用陳述句替代

### 8. ❌ 過度「請允許我」式的表述

**症狀**：
- "請允許我指出..."
- "我想強調..."
- "我想補充..."
- "允許我補充..."

**修復方案**：
```
❌ 我想強調一個重點...
✅ 一個關鍵點是...
```

### 9. ❌ 過度使用雙層語氣詞

**症狀**：同時出現兩個表達程度的詞
```
❌ 這確實非常有益
✅ 這非常有益
```

**修復方案**：
- 移除冗余的加強詞（確實、實際上、絕對、當然）
- 一句話最多一個程度詞

### 10. ❌ 模板化結論

**症狀**：用固定句式結尾
```
❌ 總之，【內容】很重要，我希望這個信息有幫助。
✅ 【具體想法或行動】
```

**修復方案**：
- 結尾用具體行動或想法，不用虛無的"希望有幫助"
- 避免所有模板化的結束句

## 清理清單

完成文檔後，逐項檢查：

### 詞彙層面
- [ ] 移除所有"革命性"、"創新"、"領先"等空心詞（除非有數據支持）
- [ ] 刪除"值得注意"、"總之"、"簡而言之"等過度轉移詞
- [ ] 移除所有"請允許我"、"我想說"式的表述
- [ ] 檢查雙層加強詞（如"確實非常"、"實際上絕對"）

### 結構層面
- [ ] 檢查是否有不必要的列表或編號
- [ ] 移除冗余的大標題
- [ ] 確保沒有連續 3 個相同句式結構
- [ ] 檢查有無重複表述相同想法的地方

### 語氣層面
- [ ] 開場白是否移除了過度正式感
- [ ] 結論是否具體而非虛無
- [ ] 全文語氣是否自然和人性化
- [ ] 是否避免了修辭問句

### 可讀性
- [ ] 平均句子長度 15-20 字
- [ ] 段落長度 3-5 句
- [ ] 段落之間無冗余過渡詞
- [ ] 讀出來是否流暢自然

## 實例對比

### 郵件範例

**❌ 有 AI 味的版本：**
```
親愛的團隊，

值得注意的是，我想向大家指出一個重要的提點。簡而言之，我們需要優化我們的流程。總之，這很關鍵。

有沒有想過，如果我們改進系統，會發生什麼？實際上，這確實非常重要。

我希望這個信息有幫助。

敬禮
```

**✅ 自然的版本：**
```
親愛的團隊，

我們需要優化流程。根據上月的數據，目前周期平均 6 週，但競爭對手的平均周期是 2 週。

如果你們有想法，週五我想聽聽。

敬禮
```

### 報告段落範例

**❌ 有 AI 味：**
```
首先，設計是革命性的。其次，實現同樣創新。最後，用戶反饋極其正面。簡而言之，該項目確實在各個方面都取得了領先的成果。
```

**✅ 自然表述：**
```
設計在用戶測試中得分 8.5/10。工程團隊在時間表內完成實現。用戶反饋指出易用性和性能是最強的兩個點。
```

## 修改流程

1. **第一遍**：移除所有虛無修飾詞和空心詞
2. **第二遍**：刪除所有"值得"、"簡而言之"式的過渡詞
3. **第三遍**：簡化結構，移除不必要的列表
4. **第四遍**：檢查語氣自然度，讀出來驗證
5. **第五遍**：確認沒有重複表述相同想法

## 何時停止清理

- ✅ 內容自然流暢，像人寫的
- ✅ 所有空心詞已移除
- ✅ 段落結構簡潔明快
- ✅ 沒有過度修飾或冗余表述
