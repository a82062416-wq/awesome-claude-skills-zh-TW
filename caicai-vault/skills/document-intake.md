# Skill：文件收件流程（Document Intake）

> 菜菜的核心工作模式：丟一份檔案給 JACK，JACK 要自己知道這是哪個專案的、該做什麼。
> 目標：越用越準，菜菜丟檔案時的說明成本趨近於零。

## 觸發時機

菜菜丟任何檔案（規章、公文、合約、報表、草稿）進對話，沒有多做說明時。

## 四步流程

### 1. 判斷所屬專案

- 讀 `projects/INDEX.md`，比對檔案內容跟進行中專案
- 對得上 → 先說「這是 XX 專案的文件」再繼續
- 對不上 → 只問一句：「這份是新專案，還是屬於哪個現有專案？」

### 2. 判斷任務類型

| 檔案狀態 | 預設動作 | 做法 |
|---------|---------|------|
| 有明顯錯誤/待改處 | **修正** | 先用表格列出問題點，再動手改 |
| 半成品 | **完成** | 先一句話確認方向，再補完 |
| 已是定稿 | **歸檔** | 放進對應專案資料夾，更新 INDEX |

### 3. 執行

輸出遵守菜菜的偏好：結論先行、表格呈現、繁體中文。

### 4. 收官

跑 `sop/after-action.md`：檔案歸位、INDEX 更新、重要結論進 memory。

## 累積規則

- 每處理一類新文件（例：勞保局公文、駐衛警排班表、社區合約），把該類文件的處理要點補進下方「文件類型筆記」，或另開專屬 skill
- 同一類文件處理到第三次還在重新解釋 → 代表筆記寫得不夠好，當場補強
